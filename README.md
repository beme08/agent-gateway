# Secure Enterprise Agent Gateway

A deployed, multi-tenant agentic AI platform with a flagship **HR Policy Agent** that answers company policy questions with ACL-filtered RAG and performs sick-leave / PTO workflows through role-gated tools. Uses Supabase RLS for tenant isolation, server-side tool authorization, prompt-injection defense-in-depth, and full audit traces.

> **Launch mode**: public demo with seeded tenants and "Try as…" role buttons. Not a real self-serve SaaS yet. See [Launch Strategy](docs/launch.md).

## Stack

- **Frontend**: Next.js 14 (App Router) + Tailwind
- **Backend**: FastAPI (Python 3.11+)
- **Database**: Supabase Postgres + pgvector + RLS
- **Auth**: Supabase Auth (email/password)
- **LLM**: Cohere `embed-english-v3.0` + `command-r-plus` (with deterministic local mock fallback)
- **Deployment**: Vercel (web) + Render/Fly.io (API) + Supabase hosted

## Quick start (local)

1. **Create a Supabase project** at https://supabase.com (free tier is fine).
2. From the Supabase dashboard, copy your project URL and keys into `.env` files:
   - `apps/web/.env.local`
   - `apps/api/.env`
3. **Run the migrations** in the Supabase SQL editor in this order:
   - `supabase/migrations/0001_init.sql`
   - `supabase/migrations/0002_rls.sql`
   - `supabase/migrations/0003_leave_functions.sql`
   - `supabase/migrations/0004_match_chunks.sql`
   - `supabase/migrations/0005_demo_reset.sql`
4. **Run the seed**:
   - `supabase/seed/seed.sql` in the Supabase SQL editor.
5. **Create demo users**:
   - `cd scripts && npm install && npx tsx create_demo_users.ts`
6. **Run locally**:
   ```bash
   npm run dev:api    # FastAPI on :8000
   npm run dev:web    # Next.js on :3000
   ```
7. Open http://localhost:3000 and click a "Try as…" button.

## Demo flow (60 seconds)

1. Open the public landing page → click **"Try as Employee"**.
2. In the chat, ask: *"I'm sick today, can you request sick leave for me?"*
3. The agent retrieves the sick-leave policy (cited), checks your balance, and calls `create_time_off_request`. You see a `pending` request and a tool-call badge.
4. Logout → click **"Try as Manager"**.
5. Open the team queue → approve the request. The balance updates and an audit event is written.
6. Logout → click **"Try as Admin"**.
7. Open the audit dashboard → click the trace to see the full timeline (retrieval → tool calls → answer).

For a more detailed walkthrough, see [`docs/demo.md`](docs/demo.md).
